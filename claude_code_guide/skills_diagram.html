📁 文件结构（M100 项目示例）

⚡ Skills 激活流程
👤 用户提问
↓
CC 发送：请求 + Skill 列表（仅 metadata）
↓
🧠 LLM 推理：选择相关 Skill
description 是触发关键，非关键词匹配
↓
CC 发送：请求 + 完整 SKILL.md
↓
🤖 LLM 响应（带完整知识）
💡 写好 description 比写好内容更重要
例：description: M100 硬件架构、sch_scope 语义、tile/memory/event 机制
🔍 渐进式披露（三层）
始终加载
元数据层：name + description
所有 Skill 的 metadata 常驻上下文，供 LLM 选择
按需加载
指令层：SKILL.md 正文内容
被选中后才加载，包含详细知识和操作指南
按需中的按需
资源层：Reference（读文件）+ Script（跑脚本）
SKILL.md 内条件触发，最深层的知识
M100 示例：/debug Skill
📄 SKILL.md → 调试方法论（指令层）
📎 Reference: cases.md → 历史案例（仅提到"案例"时读取）
🔧 Script: check.py → 地址检查（仅提到"检查地址"时运行）
🎯 自动激活
description 是触发关键，Claude 用 LLM 推理决定是否激活，不是关键词匹配。

📎 Reference & Script

Script：条件触发，直接执行脚本（只跑）

提到"检查地址" → 运行 check.py
